# Module 3: CIA Triad and Core Security Principles

## Learning Objectives

- Explain the CIA triad.
- Connect CIA to practical controls.
- Understand core principles like least privilege and defense in depth.

## CIA Triad

### Confidentiality

Ensure information is only accessible to authorized users.

**Examples:** access control, encryption, data classification.

### Integrity

Ensure information remains accurate and unaltered.

**Examples:** hashes, checksums, digital signatures, audit logs.

### Availability

Ensure systems and data are available when needed.

**Examples:** backups, redundancy, DDoS protection, monitoring.

## Supporting Principles

### Least Privilege

Users and systems should only have permissions required for their role.

### Defense in Depth

Use multiple security layers so one failure does not cause full compromise.

### Zero Trust Mindset

Never trust by default—continuously verify identity, device posture, and context.

## Practical Mapping

- MFA supports **Confidentiality**.
- File integrity monitoring supports **Integrity**.
- Disaster recovery plans support **Availability**.

## Summary

The CIA triad is a foundational model. Strong security programs balance all three areas, not just one.
